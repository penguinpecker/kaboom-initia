## 5. Reproducing this ourselves

Full launch sequence is scripted. A judge can redo the rollup in ~5 minutes:

```bash
# Prereqs: Docker, Go 1.22+, brew install initia-labs/tap/weave
brew install initia-labs/tap/weave                              # CLI
git clone --depth 1 --branch v1.2.15 https://github.com/initia-labs/minievm
(cd minievm && make install)                                    # installs minitiad

# Fund the bridge_executor L1 address with ~5 INIT
# (https://app.testnet.initia.xyz/faucet)

minitiad launch --with-config scripts/launch-config.json \
                --artifacts-dir artifacts-rollup/               # registers bridge on L1
minitiad start                                                   # blocks start producing

# Deploy Kaboom
INITIA_RPC_URL=http://localhost:8545 \
INITIA_EVM_CHAIN_ID=932743922545997 \
SEED_VAULT=100 \
npx hardhat run scripts/deploy.ts --network initia
```

That is literally how this rollup was created. `scripts/launch-config.json`
in this repo is the same config we used, minus the bridge_executor mnemonic
(which lives in our Railway secret env, never committed).
